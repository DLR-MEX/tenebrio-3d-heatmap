TENEBRIOS — Real-Time 3D Volumetric Temperature & Humidity Heatmap System
Last updated: 2026-03-27
================================================================================

PROJECT DESCRIPTION
-------------------
TENEBRIOS is a real-time IoT monitoring system that visualizes temperature
and humidity data from a mealworm (Tenebrio molitor) breeding room as an
interactive 3D volumetric heatmap. Five interior sensors (t1-t5) measure
both temperature and humidity at known (x, y, z) coordinates. An exterior
temperature sensor (tex) sits on a planter at Y=6. Readings are collected
via Ubidots MQTT, interpolated into 3D volumetric fields using scipy
griddata (nearest + linear), and rendered as Plotly.js isosurface slices.

The dashboard supports toggling between Temperature and Humidity heatmap
views via a button. Historical data can be loaded from Ubidots API and
navigated with a timeline slider.

MQTT Variables:
  Temperature:  t1, t2, t3, t4, t5 (interior sensors)
  Exterior:     tex (temperatura exterior)
  Averages:     tps (promedio superior), tpi (promedio inferior)
  Humidity:     h1, h2, h3, h4, h5, hum_general
  Ammonia:      amoniaco (PPM)
  Devices:      ventilador, extractor (on/off)

The 3D scene includes:
- Multiple isosurface slices (5 layers) filling the room with color.
  Colors use a fixed scale: blue (<21°C), green (23-28°C ideal),
  red (>30°C). cmin/cmax = 14/35.
- Toggle button: Temperature / Humidity heatmap views.
- Sensor markers (scatter3d) at their true 3D positions with drop lines.
  Labels change between t1-t5 / h1-h5 depending on mode.
- Wireframe house (10x6x3m) with inclined roof (high at Y=0, low at Y=6).
- Door on back wall (X=0, Y=2.5..3.5).
- 3 windows on Y=6 wall (equidistant, 1m wide, Z=1..2).
- 2 windows + extractor on upper Y=0 wall (Z=3.3..4.1).
- Wooden furniture below windows (X=1..9, Y=5.5..6, Z=0.3..1.0) with
  4 cabinet doors, handles, and solid mesh3d rendering.
- 2 vintage circular lamps at T1 and T3 positions with metal bars,
  circular shades, and bulbs.
- Planter + colored exterior zone at X=1..3, Y=6 (color = tex temp).
- TEX sensor marker colored by temperature.

Side panel indicators:
- 5 vertical thermometers (T1-T5) with ideal range marks (23-28°C).
- 3 horizontal temperature bars: Exterior, Superior, Inferior.
- 6 horizontal humidity bars (H1-H5, General) with ideal range marks (60-88%).
- 1 horizontal ammonia bar (0-5 PPM).
- Animated SVG fan/extractor widgets (spinning when ON).

Historical data viewer:
- Date range picker (start/end).
- Loads data from Ubidots REST API.
- Timeline slider to navigate through historical timestamps.
- Server-side interpolation for each slider position.
- "En vivo" button to return to real-time mode.

Header:
- TECHNEBRIOS logo (left) with "SUPER ALIMÉNTATE" subtitle.
- Timestamp display in Orbitron font (center).
- INGENIERIA CONDOR logo (right, CSS-rendered).
- MQTT connection status indicator.

Color palette: based on Ingenieria Condor branding.
  Primary:    #344955 (dark blue)
  Accent:     #E8B830 (gold)
  Background: #1a2630 (dark)
  Panels:     #243B4A
  Borders:    #3a5a6a
  Text:       #8aa0b0 (secondary), #E8B830 (values)

Responsive design: works on desktop, tablet, and mobile (portrait/landscape).

Camera position is preserved across data refreshes. Fetch has 5s timeout.
Auto-refresh every 2 seconds.

Temperature ideal range: 23-28°C.
Humidity ideal range: 60-88%.
Ammonia scale: 0-1 green, >2 yellow, >3 red.


================================================================================
ARCHITECTURE AND TECHNOLOGY STACK
================================================================================

Language:   Python 3.10+
Backend:    Flask (web server and JSON API)
Frontend:   Single-page HTML with Plotly.js 2.35.2 (CDN)
MQTT:       paho-mqtt v2.x (subscribe-only, Ubidots broker)
Science:    NumPy, SciPy (griddata 3D interpolation)
Testing:    pytest 7.0+
Protocol:   Ubidots MQTT /lv (last-value) subscription model
Fonts:      Google Fonts Orbitron (timestamp display)

Files:
  backend/config.py         - All configuration constants
  backend/main.py            - Application entry point
  backend/mqtt_client.py     - MQTT connection and /lv payload parsing
  backend/heatmap_engine.py  - Temperature/humidity store + 3D interpolation
  backend/visualization.py   - Flask server, API endpoints, Ubidots history
  templates/index.html       - Single-page dashboard (Plotly.js + indicators)
  static/CondorLogo.png      - Ingenieria Condor logo
  tests/test_interpolation.py - Interpolation engine tests
  tests/test_mqtt_parser.py   - MQTT parser tests
  mqtt_simulator.py          - Local test data publisher
  conftest.py                - Root conftest (adds backend/ to sys.path)

API Endpoints:
  GET /                      - Dashboard HTML page
  GET /api/data              - Real-time sensor data + interpolated volumes
  GET /api/history           - Historical data from Ubidots (start/end dates)
  GET /api/history/interpolate - Server-side interpolation for historical data
